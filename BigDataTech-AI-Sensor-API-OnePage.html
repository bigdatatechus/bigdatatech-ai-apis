Use cases

High-value APIs you can offer your customers

Failure Risk Scoring API

Given recent vibration + temp windows, return a calibrated probability of failure within a timeframe, plus confidence.

  • Outputs: risk score, time-to-failure band, confidence
  • Best for: CMMS/EAM, telematics dashboards, OEM services

Fault Type Classification API

Detect patterns consistent with imbalance, misalignment, bearing wear, looseness, lubrication issues - using your data.

  • Outputs: top fault labels, severity, evidence features
  • Best for: condition monitoring platforms

Anomaly Prioritization API

Reduce false alarms by ranking assets by urgency using business rules + model signals (criticality, downtime cost).

  • Outputs: priority queue + recommended next action
  • Best for: reliability teams managing many assets

Work Order Recommendation API

Turn model output into actionable steps: inspection type, parts checklist, and suggested scheduling window.

  • Outputs: next best action + rationale
  • Best for: CMMS automation workflows

FAQ

Common questions

Do we need an API first, or market discovery first?

In sensors/maintenance, discovery first usually wins: validate what outputs (risk score, fault label, work-order suggestion) customers will actually pay for, then build the smallest API that matches their workflow.

Can you work with limited data?

Yes. We can start with feature engineering + baselines, use public datasets for early prototyping, and design a data contract to improve continuously once deployments begin.

What does production-grade mean here?

Versioned endpoints, auth, audit logs, latency targets, monitoring/drift, and a clear plan for model updates - so your customers can rely on it.

Do you support multiple user types?

Yes: (1) end customers who send data and get predictions, (2) integrators who need tooling & docs, and (3) platform vendors who embed the API into products.
